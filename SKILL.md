# Video Director

This is a standalone Agent skill. It bundles its own minimal Video Director
runtime under `runtime/` and does not depend on an external repository layout.

## Use This Skill When

- The user wants a local activity/news/demo short video from source media.
- The user wants direct mp4 output rather than hand-editing JSON.
- The user asks whether to generate a playable video or a Jianying draft.
- The Agent needs a repeatable media-manifest -> timeline -> render workflow.

## Output Choice

Offer two user-facing paths:

1. `video`: default safe path. It renders a directly playable mp4 through ffmpeg.
2. `draft`: Jianying draft handoff. Use only when the user explicitly asks for it.

Never translate `video` into `bundle_only`, `draft_bundle.json`, or any other
intermediate artifact. A user asking for `video` means a directly playable result
video backed by the `final_render` adapter.

On macOS, tell the user Jianying draft export is not supported by default because
pyJianYingDraft is unreliable with the Mac Jianying project format. Route macOS
users to `video` unless they explicitly accept experimental draft behavior.

## Runtime Modes

Choose the smallest viable runtime mode after the user-facing output is clear:

| Mode | Use when | Primary caveat |
| --- | --- | --- |
| `local_jianying` | Local media, direct mp4, or supported local draft work | Real Jianying draft export is not the default macOS path |
| `cloud` | Explicit cloud TTS/avatar/audio delivery is needed | Requires local-only credentials and stable `job_id` for retries |
| `bundle_only` | Internal contract debugging only | Not a valid substitute for direct `video` output |

## Defaults

- Digital human/avatar is off by default.
- TTS/avatar/cloud generation is optional and must be explicitly configured.
- Subtitles are burned into the mp4 by default.
- Sidecar `.srt` is not generated by default because common players auto-load it
  and duplicate burned-in subtitles.
- `inputs.narration_text` is viewer-facing narration/subtitle text.
- `inputs.director_brief` is planning guidance and must not be displayed as subtitles.
- The skill does not ask users to configure a separate multimodal model. In Agent
  usage, the Agent inspects media with its existing model capability and writes
  or updates the assets manifest.

## Bundled Runtime

The runtime is included directly under `runtime/`. Treat that directory as an
internal implementation detail; user-facing docs and command output should say
`Video Director`.

The committed files under `runtime/templates/` are internal config templates.
Do not ask users to open, copy, or hand-edit them. Use user intent, media paths,
and any required credentials to generate a local config file during the run.

## Python Runtime

Use the cross-platform launcher when possible. It requires Python 3.11+ because
the bundled runtime is typed and validated against Python 3.11-compatible
syntax.

Unix/macOS:

```bash
bash scripts/video-director.sh --help
```

Windows:

```bat
scripts\video-director.cmd --help
```

Both launchers auto-detect a compatible interpreter. `VIDEO_DIRECTOR_PYTHON`
can be used as an explicit override when a user has Python in a custom location.

If auto-detection fails, ask the user to set:

```bash
export VIDEO_DIRECTOR_PYTHON=/path/to/python3.11
```

Use the scripts in this skill root:

- `scripts/video_director.py`: cross-platform command router.
- `scripts/video-director.sh`: Unix/macOS launcher.
- `scripts/video-director.cmd`: Windows launcher.

Use `video-director.sh` or `video-director.cmd` for normal operation. The
router calls internal modules under `runtime/`; there are no separate
user-facing CLI helpers to run directly.

## Workflow

### 1. Understand Materials

For Agent usage, inspect the source media yourself and write a structured manifest
that includes paths, tags, descriptions, scene types, mood, and best-use hints.

For a deterministic offline fallback:

```bash
bash scripts/video-director.sh analyze \
  --materials-dir /path/to/source-media \
  --output /path/to/workspace/assets_manifest.json
```

The offline analyzer uses filename/path hints only. Improve the manifest with
Agent visual understanding when quality matters.

### 2. Prepare Config

Generate a local config from the internal templates. The user should provide
only natural-language intent, media locations, output preference, and any
credentials required by an optional path. Do not make the user copy JSON from
template files.

Default direct-mp4 config command for the Agent:

```bash
bash scripts/video-director.sh config local_jianying \
  --output-mode video \
  --output /path/to/workspace/video-director.video.local.json \
  --job-id demo-video \
  --narration-text "这里填写面向观众的旁白字幕" \
  --director-brief "这里填写剪辑方向，不会作为字幕显示" \
  --set production.assets_manifest_path='"/path/to/workspace/assets_manifest.json"' \
  --set production.full_tts_duration_ms=30000 \
  --set outputs.final_render.output_name='"demo-video.mp4"'
```

Use `--set` for advanced options such as subtitle style, BGM/audio tracks, output
root, transition duration, or explicit sidecar subtitles.

Read `references/config_contract.md` before adding new config keys.

### 3. Doctor

```bash
bash scripts/video-director.sh doctor /path/to/workspace/video-director.video.local.json
```

Stop on required errors. `ffmpeg` is required for real mp4 rendering.

### 4. Dry Run Then Render

```bash
bash scripts/video-director.sh run /path/to/workspace/video-director.video.local.json --dry-run
bash scripts/video-director.sh run /path/to/workspace/video-director.video.local.json
```

The script resolves relative media and output paths against the caller's working
directory. Set `VIDEO_DIRECTOR_WORKSPACE_ROOT=/path/to/workspace` when running
from another directory.

### Contest Demo

For a local no-copyright demo:

```bash
bash scripts/video-director.sh demo
bash scripts/video-director.sh doctor demo/contest/video-director.contest-demo.local.json
bash scripts/video-director.sh run demo/contest/video-director.contest-demo.local.json --dry-run
bash scripts/video-director.sh run demo/contest/video-director.contest-demo.local.json
```

### 5. Summarize

```bash
bash scripts/video-director.sh summarize output/video_director/<job_id>/latest_run.json
```

Report the mp4 path, duration, whether avatar/audio tracks are present, subtitle
cue count, and whether a sidecar `.srt` exists.

## Failure Handling

- If `doctor` fails on Python, ffmpeg, or Pillow, fix the local runtime first.
- If `video` rendering fails, inspect `final_render.render_plan.json` and the
  ffmpeg error.
- If a sidecar `.srt` appears unexpectedly, check `outputs.final_render.emit_sidecar_srt`.
- If subtitles show director text, check that narration is in `inputs.narration_text`
  and planning text is in `inputs.director_brief`.
- If material matching is weak, improve the manifest instead of adding a separate
  model configuration step.
- If draft export reports missing `pyJianYingDraft`, do not silently downgrade to
  a bundle file. Install the optional dependency or explain that real draft
  generation is unavailable in the current environment.

See `references/troubleshooting.md` only when a run actually fails.
